Frequently Asked Questions

Yes, 100% safe. SecuriScan runs entirely inside your browser — no data is sent to any external server. All analysis is local, we have no backend, and we collect zero telemetry. The full source code is open on GitHub so you can verify this yourself.
SecuriScan uses Manifest V3 and works on:
  • Google Chrome (v88+)
  • Microsoft Edge (Chromium-based)
  • Brave Browser
  • Any other Chromium-based browser
Firefox support is not available yet, but is on the roadmap.
A greyed-out icon typically means the extension cannot access the current page. This happens on:
  • Chrome internal pages (chrome://)
  • New tab / blank pages
  • Chrome Web Store pages (browser restriction)
  • PDF files opened in the browser
Navigate to a regular https:// website and it will activate normally.
The score (0–100) is a weighted average across 8 security categories. A lower score doesn't mean a site is actively malicious — it means security best practices aren't fully implemented. Think of it as a posture indicator, not a definitive verdict. Always review individual findings for context.
Common reasons:
  • Missing security headers — Add Content-Security-Policy, Strict-Transport-Security, etc. via your server or CDN.
  • Outdated libraries — Update jQuery, Lodash, etc. to current versions.
  • Cookies without flags — Set HttpOnly and Secure on session cookies.
  • Mixed content — Ensure all resources (fonts, scripts, images) load over HTTPS.
The extension report includes specific recommendations for each finding.
SecuriScan is a passive analyzer only — it never makes intrusive requests, modifies traffic, or attacks anything. It's great for a quick security health check but is not a replacement for tools like Burp Suite or OWASP ZAP for active penetration testing. Always obtain proper authorization before testing any system you don't own.
After scanning a page, scroll to the bottom of the SecuriScan popup and click "Export HTML Report." A styled HTML file will be downloaded containing all findings, scores, and recommendations — ready to share with your team or clients.
The latest version is always available on the Chrome Web Store. Updates are delivered automatically by Chrome — you don't need to do anything manually.

Report a Bug

To help us fix issues quickly, please include the following information in your GitHub issue:

  • 1
    Chrome version — found at chrome://version/
  • 2
    SecuriScan version — shown at the top of the extension popup
  • 3
    URL of the page where the issue occurred (if not sensitive)
  • 4
    Steps to reproduce — what you did, what you expected, what happened instead
  • 5
    Console errors (optional) — open DevTools → Console → copy any red errors
